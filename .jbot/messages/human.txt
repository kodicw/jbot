ARCHITECTURAL DIRECTIVE: 
1. INTERNAL COHESION: JBot components (infra, ops, etc.) must remain under a single Linux user account. Do NOT attempt to split a single organization across multiple users.
2. EXTERNAL ISOLATION: Multi-project management is handled exclusively by Home Manager/NixOS using separate user accounts for entirely different organizations (Project A = User A, Project B = User B).
3. HIERARCHY REMOVAL: All internal sub-project, nested project, and hierarchical scaling logic has been REMOVED. Do NOT re-implement these features. JBot is now a flat multi-agent organization.
